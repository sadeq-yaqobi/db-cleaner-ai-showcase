Frequently Asked Questions

Find answers to common questions about DB Cleaner AI

General Questions

What is DB Cleaner AI?

DB Cleaner AI is a WordPress plugin that uses artificial intelligence to analyze your database tables, identify which plugin created them, and help you safely remove unnecessary tables to optimize your database.

How does DB Cleaner AI help my WordPress site?

The plugin helps improve database performance and reduce bloat by identifying and removing unused or orphaned tables left behind by uninstalled plugins. This can lead to faster database queries, reduced backup sizes, and better overall site performance.

Is DB Cleaner AI compatible with my WordPress version?

DB Cleaner AI is compatible with WordPress 5.0 and above. We recommend always using the latest version of WordPress for optimal security and performance.

Is this plugin free or premium?

DB Cleaner AI is a premium WordPress plugin that requires purchase before use. All features described in this documentation are included in the purchased version of the plugin. There is no free version available at this time.

Can I use DB Cleaner AI on multisite installations?

No, DB Cleaner AI doesn't support WordPress multisite installations.

Setup and Configuration

How do I get a Hugging Face API key?

  1. Create an account at Hugging Face
  2. Navigate to your profile settings at https://huggingface.co/settings/tokens
  3. Generate a new API token (Read access is sufficient)
  4. Copy this token and paste it into DB Cleaner AI settings

Which AI model should I choose?

We recommend starting with the default Mixtral-8x7B-Instruct model, which offers a good balance of accuracy and performance. If you're experiencing slow analysis times, you can switch to a smaller model like Gemma-2-2B or DeepSeek-1.5B.

Are there any additional costs to using this plugin?

You just buy it once, but you may incur charges from Hugging Face if you exceed their free API usage limits. Most small to medium-sized WordPress sites will stay within the free tier limits with normal usage patterns.

How often should I run the analysis?

We recommend running a full database analysis after:

  • Major WordPress updates
  • Installing or removing several plugins
  • At least once every 3-6 months as regular maintenance

AI Analysis

How accurate is the AI analysis?

The AI analysis is highly accurate for identifying common WordPress plugin tables (80-95% accuracy). However, for custom or obscure plugins, the accuracy may be lower. Always review the AI recommendations carefully before taking action.

Why are some tables not identified?

Some tables may not be identified for several reasons:

  • Custom-built plugins with unique naming conventions
  • Tables created by developers without standard patterns
  • Very old plugin tables from discontinued plugins
  • Tables with generic names that don't follow standard conventions

How does the plugin determine if a table is safe to delete?

The safety assessment is based on several factors:

  • Whether the table matches WordPress core tables
  • If the table is currently being used by active plugins
  • The presence of recent data modifications
  • Relationships with other database tables
  • Known patterns of essential plugin tables

Security and Privacy

Is my data secure when using DB Cleaner AI?

Yes. DB Cleaner AI only sends table names (with prefixes removed) and column structures to the AI service. No actual data from your database is ever transmitted or analyzed. We take security seriously and follow WordPress best practices for all operations.

What exactly is sent to the AI service?

Only the following information is sent:

  • Table names (with database prefix removed)
  • Column definitions (name, type, key information)
  • Active plugins name

Here's an example of what might be sent:

Table: posts
Columns: 
- ID (bigint, Primary Key)
- post_author (bigint)
- post_title (text)
- post_content (longtext)
...

Can I use DB Cleaner AI on sites with sensitive data?

Yes. Since no actual table content is ever transmitted, the plugin is safe to use on sites containing sensitive or personal data. We comply with GDPR and other privacy regulations.

Does the plugin create backups before deletion?

The plugin does not automatically create backups. We strongly recommend manually backing up your database before deleting any tables. You can use plugins like UpdraftPlus, BackWPup, or your hosting provider's backup tools.

Table Management

Can I recover deleted tables?

No. Once a table is deleted, it cannot be recovered unless you have a database backup. Always back up your database before performing any deletions.

Why are some tables marked as "Disabled-Undeletable"?

Tables marked as undeletable include:

  • WordPress core tables (wp_posts, wp_users, etc.)
  • Tables created by DB Cleaner AI itself (for logging purposes)

What does the red border around some tables mean?

Tables with a red border are identified as potentially critical tables that should not be deleted. These typically include:

  • Tables containing essential plugin configuration
  • Tables storing user information or content

What does the yellow border around some tables mean?

Tables with a yellow border require caution before deletion. These are typically:

  • Tables from plugins that may still be active
  • Tables that might have relationships with other active plugins
  • Tables where the AI has medium confidence in its assessment

Can I exclude tables from analysis?

Currently, the plugin doesn't have an exclusion list feature. However, you can simply choose not to analyze certain tables by not selecting them in the interface.

Troubleshooting

The API connection test is failing

Try these steps:

  1. Verify your API key is correct (no extra spaces)
  2. Check your internet connection
  3. Visit the Hugging Face status page to verify their service is up
  4. Try a different AI model
  5. Check if your server firewall is blocking outbound requests

The plugin is analyzing very slowly

To improve analysis speed:

  1. Select fewer tables at once (5-10 maximum)
  2. Choose a smaller/faster AI model like Gemma-2-2B
  3. Check your server resources (CPU, memory)
  4. Increase PHP memory limit if possible
  5. Try during off-peak hours when your server has more resources available

I deleted a table and now my site has errors

If you're experiencing issues after deleting a table:

  1. Restore from a backup immediately if available
  2. Check your WordPress error logs for specific errors
  3. Contact our support team with details about the deleted table and the errors you're seeing

The plugin seems to be using a lot of server resources

AI analysis can be resource-intensive. To minimize impact:

  1. Analyze fewer tables at once
  2. Use smaller AI models
  3. Schedule analysis during low-traffic periods
  4. Consider upgrading your hosting if you have limited resources

Technical Questions

How does the plugin handle multisite installations?

On multisite installations, DB Cleaner AI will:

  • Show tables specific to each subsite when accessed from that subsite's admin
  • Show all network tables when accessed from the network admin
  • Keep separate logs for each subsite
  • Respect network administrative boundaries

Can I exclude certain tables from analysis?

Currently, there's no built-in exclusion list, but we're planning to add this feature in a future update. For now, you can simply avoid selecting tables you don't want to analyze.

Does the plugin work with custom table prefixes?

Yes. DB Cleaner AI automatically detects and works with custom table prefixes, ensuring proper identification and analysis regardless of your prefix setting.

How much server resource does the analysis process use?

The analysis process primarily uses API calls to external AI services, so the local resource usage is minimal. However, preparing large amounts of table data for analysis can temporarily increase memory usage. If you have a large database (100+ tables), consider analyzing in smaller batches.

What happens if my API key expires?

If your API key expires or becomes invalid, the plugin will display an error message when you attempt to perform an analysis. Simply generate a new API key and update it in the settings.

Can I export the analysis results?

Currently, the plugin doesn't have a built-in export feature, but we're planning to add CSV/PDF exports in a future update. For now, you can use browser print functions or screenshots to save analysis results.

How does DB Cleaner AI handle tables with foreign key relationships?

The AI analysis attempts to identify relationships between tables, including foreign key constraints. When such relationships are detected, the plugin will recommend caution if deleting one table might affect others.

Can I schedule automatic analyses?

Currently, the plugin doesn't support scheduled analyses, but this feature is on our roadmap for future releases. For now, all analyses must be manually triggered.
